# 个人记账系统 C++ 控制台版
## 项目简介
基于C++控制台开发的本地记账工具，使用vector内存存储账单数据，搭配bill.txt文本文件持久化存储，完整实现账单新增、修改、删除、收支统计、本地文件读写；Windows控制台自动清屏，全链路异常捕获防止程序闪退。

### 新增拓展特性（迭代更新）
1. 钱包初始本金机制：自定义启动本金，当前余额 = 初始本金 + 所有账单收支总和
2. 账单日期记录：新增日期字段，支持自定义日期录入
3. 日期标准化：输入简写格式`2026.1.5`自动补零统一存储为`2026.01.05`
4. 自动排序展示：查看账单时临时按日期升序，同日按创建底层ID先后排序，原始文件存储顺序不变
5. 账单修改功能：可按展示序号修改账单名称、金额、日期

### 核心设计特点
1. **双层ID分离机制**
- 用户可见连续展示序号：删除账单后自动补齐，永远无断层，操作直观
- 底层永久唯一ID：仅存于bill.txt，每条账单终身独有，只会递增不会复用，用于底层溯源，终端界面不展示
2. 数据持久化：程序启动自动读取历史账单，退出可选择保存写入本地文件
3. 全流程异常拦截：非法字符输入、文件读写失败、越界删除、空名称、非法日期均捕获，程序不会闪退
4. Windows控制台自动清屏，操作完成按回车刷新菜单，界面整洁
5. 面向对象分层：Date日期基类、bill账单子类、accountbook账本管理类，代码解耦易拓展

## 功能菜单
=====个人记账系统=====
【当前钱包余额：实时计算总额】
1.新增账单
2.查看全部账单（自动按日期排序，展示序号永久连续）
3.根据展示序号删除账单
4.收支统计
5.保存账单到文件
6.修改钱包初始本金
7.按展示序号修改账单信息
0.退出程序

### 功能详细介绍
1. **新增账单**
输入账单名称、自定义日期（格式支持`2026.1.1`简写，自动补零为`2026.01.01`，直接回车使用今日标准化日期）、金额（正数=收入，负数=支出）；自动分配底层永久唯一ID存入容器。

2. **查看全部账单**
程序内部临时拷贝账单列表自动排序展示：日期由早到晚，日期相同则按创建先后；打印连续无断层展示序号、标准化日期、名称、金额；原始存储顺序不会改变，bill.txt内保持录入顺序。

3. **删除账单**
进入功能自动打印全部账单，输入界面展示的连续序号执行删除；删除后后续账单展示序号自动前移补齐，无断层。

4. **收支统计**
自动计算总收入、总支出、账单净差额、钱包初始本金、当前实际钱包余额。

5. **保存账单到文件**
将当前初始本金、全部账单写入bill.txt持久化存储，日期统一为`yyyy.MM.dd`标准格式。

6. **修改钱包初始本金**
修改钱包最开始持有的本金基数，当前余额会自动重新计算（余额=本金+全部账单收支）。

7. **修改账单信息**
先展示全部账单，输入目标展示序号，可选修改：1账单名称 / 2账单金额 / 3账单日期；修改金额会自动同步计算总余额，修改日期自动标准化补零。

8. **退出程序**
可选 0取消退出 / 1先保存再退出 / 2直接退出不保存。

## 编译运行教程
### 运行环境：Dev-C++ Windows
1. 双击`作业.dev`打开项目工程
2. 确认项目包含`main.cpp`、`test.h`两个核心代码文件
3. 顶部菜单栏「运行」→「编译运行」，自动生成`作业.exe`可执行程序
4. 首次运行同目录自动生成`bill.txt`空白数据文件存储账单
### 如果采用非dev来运行，则需要对应的编译器通过终端编译实现

### 系统适配说明
- 文件统一保存编码：UTF-8，控制台中文无乱码

## 数据文件 bill.txt 存储格式
第一行数字：钱包初始本金
后续每行4项数据，空格分隔：底层唯一ID 标准化日期 账单名称 金额
示例：
1000
1 2026.06.01 工资 6000
2 2026.06.02 午餐 -25.5
3 2026.06.03 打车 -12
> 注意：不建议手动修改/删除txt文件，非法格式会导致程序读取异常，所有账单操作请通过程序完成。

## 异常容错机制
1. 菜单输入字母、符号等非数字，自动拦截提示，返回菜单不闪退
2. 删除/修改不存在的账单序号、空账单名称、非法日期格式给出友好提示
3. 文件打开失败、读写异常捕获，打印错误信息，程序正常运行不崩溃
4. 顶层全局try-catch兜底捕获未知致命异常，保障程序安全退出
5. 读取文件时自动标准化日期，兼容旧文件简写日期数据

## 注意事项
1. 直接删除bill.txt会清空所有本金与账单记录，程序重启后重建空白文件
2. 编译缓存`main.o`、布局文件`作业.layout`可直接删除，不影响程序业务逻辑
3. 底层真实ID只会持续递增，删除账单不会复用旧ID，保证每条账单数据全局唯一
4. 代码分层解耦：日期逻辑独立Date类、账单数据独立bill类、账本管理逻辑独立accountbook类，拓展性强
5. 余额计算逻辑：不单独存储余额，每次实时计算「初始本金+所有账单总和」，数据无偏差
